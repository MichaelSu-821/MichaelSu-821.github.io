部分软件、框架操作记录

发表于 2021-10-10 更新于 2021-12-01
即将步入工作，主要记录自己在折腾各种软件的过程中的一些操作记录，方便后续再次使用作为参考。

conda

  1. 查看所有虚拟环境：conda info -e

  2. 删除虚拟环境：conda remove -n name --all

  3. 环境回滚：查看所有的记录conda list --revisions，恢复到指定的版本conda install --revision [revision number]

Git

  1. 修改本地配置：git config --global user.name "name"，git config --global user.email "email-address"
  2. ssh-keygen -t rsa -C “xxx@gmail.com“
  3. 拷贝 ~/.ssh 中的id_rsa.pub中的内容到github中设置内的SSH公钥部分

Hexo搭建博客

总体参考这篇博客：https://kaiter-plus.gitee.io/2020/03/07/How_To_Freely_Build_Blog/

我碰到的问题：

  1. 2.2.搭建中，第一步应该是安装cnpm(猜测类似于国内镜像，后续涉及到npm install的内容可以使用cnpm install代替)，另外正确命令应该是npm install -g cnpm --registry=https://registry.npm.taobao.org；
  2. 4.5.上传到gitee中，关于_config.yml的配置，如果你在Gitee中添加了ssh公钥，建议在repo:部分使用仓库的ssh地址(使用Http可能需要配置用户名和密码等内容？我没有具体尝试，至少我使用Http没有成功，后续研究明白后再来更新)；
  3. hexo init后提示一个Warring，具体没有记录，随后执行hexo s会报错，然后提示WARN Failed to install dependencies. Please run ‘npm install’ manually!。输入cnpm install，多输入几次，直到不报错为止;
  4. Gitee Pages同步还有问题，不知道是内容没有推送上去还是部署之类的有问题，主题一直显示有问题;
  5. 主题为Hexo-Next主题，网上搜到的教程大部分基于Next-v5，现在应该是v8了，有许多改变，暂时不折腾了，作为一个记录贴，后续有精力再来进行美化之类的操作。

搭建图床插入图片

  1. 使用的软件：VS Code + 插件PicGo

  2. 设置

    1. Github创建仓库并申请令牌

      Settings->Developer settings->Personal access tokens->Generate new token

    2. PicGo绑定仓库

      Token中填入申请的令牌，申请令牌时勾选repo、admin:repo_hook、delete_repo(具体什么意思还没有研究)

    3. 截图，Ctrl+Alt+U上传图片到仓库并自动粘贴链接

  3. 关闭VS Code的Security Mode，否则在no trust workspace中这些插件都无法使用，至于怎么将workspace设置为Trusted，目前还没研究明白。
