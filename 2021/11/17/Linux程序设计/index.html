括号的作用

参考链接

()

  • 命令替换，$(cmd)结构，执行其中的cmd命令
  • 初始化数组，array=(a b c d)
  • 命令组，新开一个shell执行括号内的多条命令，命令之间用分号隔开

(())

  • flag=((exp)) exp=0—flag=1,exp=1—flag=0
  • $((exp)) 计算结果，自动转换为10进制
  • ((a++)) a自加1
  • 相当于之前的算术比较部分 exp1 -eq exp2 —- ((exp1 == exp2)) ...

[]

  • array中用来表示下标，引用每个元素
  • 等同于test，将参数表达式作为比较表达式或者作为文件测试，test的退出码决定是否需要执行后面的代码，while循环的条件可以改写成while test "trythis" != "secret"。test和[]中只能用!=或者==
  • 正则表达式，描述一个字符的匹配范围

[[]]

  • 作为条件判断比[]更强，可以使用&&\||\<\>。if [[ $a != 1 && $a != 2]] ---- if [ $a -ne 1 ] && [ $a != 2]
  • 测试发现，#!/bin/sh的条件下不能使用[[]]，#!/bin/bash的条件下可以使用

{}

  • touch {a..d}.txt————–result:a.txt, b.txt, c.txt, d.txt
  • 在只允许使用单条语句的地方构建一个语句块，可以执行多条语句
  • 几种特殊替换结构
    • ${var:-string},${var:=string}
      • same: var为空时，${var:-string}=${var:=string}=$string;var不为空时，${var:-string}=${var:=string}=$var
      • differe: var为空时，${var:-string}=$string; var=string；${var:=string}，var不变
    • ${var:+string} var不为空时，使用${var:+string}=$string，否则${var:+string}=$var，即为空
    • ${var:?string} var不为空时，使用${var:?string}=$var，否则将string输出到标准错误，并从脚本退出
  • 匹配替换结构
    • 在键盘的布局上#在%左边，所有#从左边开始删，而%从右边开始删
    • 单字符最小匹配，双字符最大匹配
      • # 从头开始删，最小匹配 ${var#string}
      • ## 从头开始删，最大匹配 ${var##string}
      • % 从尾开始删，最小匹配 ${var%string}
      • %% 从尾开始删，最大匹配 ${var%%string}

命令

  1. eval
  2. exec –将当前shell替换为另一个shell
  3. export –export后的变量可以在调用的另一个脚本中使用
  4. expr –等同于$(())
  5. set –将命令的输出设置为shell的参数
  6. shift –将shell的参数变量左移一位，即shift之后，新的$1为原来的$2
  7. trap –trap [cmd] [signal]，捕获到signal后执行cmd
  8. unset –从环境中删除变量或函数(unset IFS--相当于恢复IFS的默认值？)
